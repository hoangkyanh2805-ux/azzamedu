# Framework - Dead Cycle To Codified System

> Source inspiration: NCI founder/bootcamp case study.  
> Reusable for: trading education, AI automation, content systems, sales playbooks, high-ticket coaching, cohort bootcamps.

## Purpose

Turn scattered learning or repeated failed attempts into a repeatable system that can be taught, improved, and sold.

## The Pattern

```text
Pain
-> Search
-> Dead Cycle
-> Cornerstone
-> Case Library
-> Backtest / Replay
-> Standardize
-> Integrate
-> Live Confirm
-> Proof
-> Teach
-> Community
-> Cohort
```

## Definitions

| Step | Plain Meaning | Failure Mode |
|---|---|---|
| Pain | A real problem forces the person to seek change | Weak pain creates weak follow-through |
| Search | The person studies many options | Consuming more than practicing |
| Dead Cycle | Search -> try -> fail -> search again | Strategy hopping |
| Cornerstone | One trusted foundation to build around | Picking a weak or untested foundation |
| Case Library | Many examples/cases collected for review | Too few examples, cherry-picking |
| Backtest / Replay | Test the idea on past cases | Looking only for confirmation |
| Standardize | Turn intuition into rules/checklists | Rules too vague to repeat |
| Integrate | Add supporting concepts only after the base works | Adding complexity too early |
| Live Confirm | Test in real conditions | No journal, no feedback loop |
| Proof | Evidence that the system works | Unverified or exaggerated claims |
| Teach | Package the system for others | Teaching before the method is stable |
| Community | Shared practice, support, identity | Group becomes hype instead of feedback |
| Cohort | Time-bound guided implementation | No cadence, homework, or review |

## Diagnostic Questions

Use these when analyzing any offer, course, or founder story:

1. What is the prospect's pain?
2. What failed search loop are they stuck in?
3. What is the named enemy or mistake?
4. What is the cornerstone mechanism?
5. What case library or evidence created the rules?
6. Which rules are standardized?
7. Which concepts were integrated later?
8. What counts as live confirmation?
9. What proof is personal, student, or third-party verified?
10. What delivery container turns knowledge into execution?

## Offer Template

```text
For people stuck in <dead cycle>,
we teach <cornerstone mechanism>
through <guided container>
so they can build <repeatable system>
without <common failure mode>.
```

Example for compliant trading education:

```text
For traders stuck in random setup hopping,
we teach a risk-first operating desk
through a guided course, journal workflow, and weekly review rhythm
so they can make fewer impulsive decisions
without relying on signals or profit promises.
```

## Landing Page Structure

```text
1. Identity hook: who this is for
2. Pain mirror: what they are tired of
3. Dead cycle diagnosis: why they stay stuck
4. Mechanism: the cornerstone system
5. Method: case review, rules, homework, feedback
6. Proof: process proof + verified outcomes where available
7. Container: course, community, mentor, cadence
8. Risk boundary: what this is not
9. CTA: join/start/apply
```

## Agent Loop

```text
observe source -> extract pain/dead-cycle/mechanism/proof
-> classify proof confidence
-> map offer ladder
-> flag compliance risks
-> generate reusable framework
-> produce landing/offer checklist
```

## Compliance Checklist For Trading Offers

- [ ] No guaranteed profit language.
- [ ] No implication that funded-account results are typical.
- [ ] Proof is labeled: source claim, platform verified, audited, or anecdotal.
- [ ] Drawdown/risk language is present near result claims.
- [ ] Offer promises education/process, not account management.
- [ ] Community is positioned as support, not signals.
- [ ] CTA does not pressure users with unrealistic income claims.
